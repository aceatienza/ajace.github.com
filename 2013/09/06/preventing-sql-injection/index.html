Preventing SQL Injection

SQL injection is the act of inserting sql queries as input by an attacker. Take a user login form as an example. Instead of inserting a username for the login, a hacker could put in SELECT * FROM USER; retrieving all information from the Users table.

To prevent this, user input should be sanitized (should be sanitized all the time anyways),

Use prepared statements. [Example using PDO](link to asana report database) and the older Example using mysqli

Frameworks will most likely provide this functionality for you. An example, the [Asset Tool] I recently built in Laravel is protected from SQL injection because Laravel’s Fluent query builder and Eloquent ORM use PDO parameter binding. [link to Laravel api with PDO]
