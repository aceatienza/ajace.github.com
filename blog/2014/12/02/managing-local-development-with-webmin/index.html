Managing local development environment with Webmin (Linux)

At Vector Media, I use Mamp Pro for local dev since it has the ability to set individual versions of PHP for a couple of the sites that I work on. At home, I’ve gotten tired of manually editing php.ini, the virtual hosts file, and my hosts file and just want an easy to use control panel. Here’s a good list. I was going to go with Ispconfig which looked the most polished but it seemed like overkill after looking at the screenshots so I went with Webmin.

Install Webmin 

sudo add-apt-repository 'deb http://download.webmin.com/download/repository sarge contrib'  
sudo add-apt-repository 'deb http://webmin.mirror.somersettechsolutions.co.uk/repository sarge contrib'  
wget http://www.webmin.com/jcameron-key.asc  
sudo apt-key add jcameron-key.asc  
sudo apt-get update  
sudo apt-get install webmin

(optional) Apply a theme - download locally and install from Webmin Configuration > Webmin Themes activate at Webmin > Change Language and Theme

(optional) Install module phpMyAdmin link - download locally and install from Webmin Configuration > Webmin Modules

Then fire up Webmin by going to https://localhost:10000/

Note: if you’re using Chrome it will complain about an invalid certificate (other browsers may also give this warning). Just click through or you can fix the issue - FAQ. Also in the FAQ is how to allow remote connections under “How can I change Webmin’s list of allowed IP addresses from the shell?”

I already had my LAMP stack setup so I didn’t have to do any additional configuration in Webmin.
