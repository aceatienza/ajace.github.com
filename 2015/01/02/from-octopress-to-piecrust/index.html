From Octopress to Piecrust

Templating: Liquid vs Twig Server: generate, watch, preview

Missing built in SASS/LESS pre-processing so you’ll have to setup your own or extend the generate command to include compilation. There’s also Piecrust.

There are no command line helpers currently to easily generate a new post or a page ala “rake newpost” or “rake newpage”. Just create an empty template and simply copy it each time with a dashed date in front.

cp post_template.md 2015-01-01-spicy-bacon-is-good-on-everything.md
  • possible to auto insert date from file name using Twig? Permalinks are inconsistent.

permalink: blog/:year/:basename

would often times stop working

Dates are formatted using Twig http://twig.sensiolabs.org/doc/filters/date.html

Slight change to the file structure. Copied all posts over from Octopress into /_content/posts, then changed the extension and the dash after the date to an underscore.

rename -v 's/\.markdown/\.html/' *.markdown
rename -v 's/(.{10})(.)/$1\_/' *.html

Piecrust determines the date from the filename and you must remove the date variable from the yaml header.

There is also ‘chef import’ which I didn’t try that might work.

Yaml header in the page or posts configuration is sensitive to tabs so it seems to always be better to use spaces. I use Sublime Text with the indent replacement setting to 4 spaces.
