Vim Cheatsheet

These are the vim keys and commands which I have been using the most for web development (PHP, HTML, CSS, Javascript) and sys admin. Also, here’s my .vimrc

FILES

:e scp://user@host/path/file * open remote file in a local VIM :ls (or :buffers) list of open files/buffers
:bn next buffer :bp previous buffer :b# (# is a number to move to, same as switching open windows in Ubuntu 13.04 with ALT+1, ALT+2, etc.) :bw buffer wipe (delete buffers) :e open a new buffer (open a new file and load into buffer) :b TAB cycle through the open files/buffers list

SESSIONS

:mksession! ~/today.ses to save the session and call it today vim -S ~/today.ses to reload the previous session

TABS

:tabe [filename] open file in new tab :tabc close :tabn, :tabp switch tabs

WINDOWS

Ctrl-E scroll down Crtl-Y scroll up

Ctrl-W s and Ctrl-W v to split the current window horizontally and vertically.

Ctrl-W w to swhitch between open windows, and Ctrl-W h (or j or k or l) to navigate through open windows.

Ctrl-W c to close the current window, and Ctrl-W o to close all windows except the current one

INDENTATION

== align the current line =i{ align the inner block =% align to the matching parenthesis/bracket under the cursor 14== align the next 14 lines =G align to the end of the file

SEARCH

/word will search forward word. add :set wrapscan in .vimrc so that this is the only thing you’ll need ?word will search backward for word n next N previous ?word reverse search for word * will search for the next instance of the word under the cursor

will search for the previous instance of the word under the cursor

GETTING AROUND IN THE FILE

G jump to end of buffer gg jump to beginning of buffer

G or #gg jump to # (line number)

COPY, CUT, PASTE, UNDO

  • select text in visual mode then y yank (copy) d same as cut p paste after P paste before

HISTORY

gi jump back to last place you were in Insert mode u undo Ctrl-R redo

FOLDING

  • I don’t fold stuff

AUTOCOMPLETION

Ctrl-x Ctrl-o, then Ctrl-n to go select next suggestion

EXTRA NOTES

  • you can’t copy from a remote session to the clipboard
  • commands have auto-complete

Wordpress: send email on published post

The NYC Huskies site needs to send out an email of an author, typically Katie, to our alumni listserv. It was previously sent using Subscribe2 but, since we don’t really use the subscription function for the site, it’s kind of overkill to use a plugin just for one simple function. The first version sends an email only on the original publication. The second sends an email on post updates.

Send using wp_mail() and then add_action()

“` php functions.php function notifyviaemail( $postid ) { $post = getpost( $post_id ); $to = ‘example@example.org’; 

        $subject = get_bloginfo('name') . $post->post_title;

        <-- note: you can also build an html email -->
        $message = $post->post_title . ' was published on ' . get_bloginfo( 'name' ) . ' as of ' . $post->post_date . '. You may view it at ' . get_permalink( $post_id ) . '.';

        wp_mail( $to, $subject, $message );
    }
    add_action('draft_to_publish', 'notify_via_email');
    add_action('pending_to_publish', 'notify_via_email');
    add_action('new_to_publish', 'notify_via_email');

Properly installing Disqus on Wordpress

I learned this the hard way when installing on NYC Huskies. To properly install Disqus on Wordpress, you must register your site before you install the plugin.

If you do make the mistake of activating the plugin, to reset, go to “Plugin Settings” and scroll down to “Reset”. Deactivate and delete the plugin and redownload and install.

Unfortunately for me, it took the shortname for this site and I had to use a slightly different one. I’m not sure if Disqus releases these short names and I couldn’t find any mention on Disqus’ support pages on how to claim.
